# 第一階段：數據降臨 (Data Ingestion & Inspection)
目標： 快速確認資料庫抓下來的東西「長什麼樣子」。

核心功能：
- pd.DataFrame(data): 建立資料框。
- df.shape: 看資料有幾列幾行（SA 習慣先看 Data Volume）。
- df.info(): 檢查有沒有欄位是 NULL，或是型別錯誤（例如日期變成字串）。
- df.head(n): 觀察前 n 筆資料的規律。

## 任務 1：檢查機台清單
1. 請讀取一組包含 Machine_ID, Status, Efficiency 的資料，並找出這份資料一共有多少筆記錄。

# 第二階段：數據過濾與清洗 (Data Cleaning)
目標： OI 工程師最常做的事——排除異常值、只看特定產線。

核心功能：
- 條件篩選：df[df['Efficiency'] < 0.8] (找出低效能機台)。
- 多重條件：使用 & (AND) 與 | (OR)。
- 缺失值處理：df.fillna() 或 df.dropna()。
- 欄位轉換：pd.to_datetime(df['Date']) (這在處理 Time-series 時是必考題)。

## 任務 2：找出異常機台
1. 過濾出所有狀態為 Offline 且 Efficiency 低於 0.5 的機台。
2. 將所有效率為 NaN (缺失) 的數值填補為該欄位的平均值。

# 第三階段：聚合與洞察 (Aggregation & Insight)
目標： 把散亂的細節變成主管要看的「戰情報表」。

核心功能：
- 分組計算：df.groupby('Department')['Output'].sum() (按部門算總產出)。
- 多重聚合：.agg(['mean', 'max', 'count']) (一次看平均、最大值與數量)。
- 排序：df.sort_values(by='Efficiency', ascending=False)。

## 任務 3：部門效能報表
1. 依照 Department 分組。
2. 計算每個部門的 平均效率 與 總機台數量。
3. 按照平均效率由高到低排序，找出表現最差的部門。

```python
Python
import pandas as pd
import numpy as np

# 模擬 OI 原始數據
data = {
    'Machine_ID': ['M01', 'M02', 'M03', 'M04', 'M05', 'M06'],
    'Dept': ['Etch', 'Etch', 'Photo', 'Photo', 'ThinFilm', 'ThinFilm'],
    'Efficiency': [0.85, 0.42, 0.91, np.nan, 0.77, 0.35],
    'Status': ['Online', 'Offline', 'Online', 'Online', 'Online', 'Offline']
}

df = pd.DataFrame(data)

# --- 開始您的任務 ---
# 1. 填補缺失值
df['Efficiency'] = df['Efficiency'].fillna(df['Efficiency'].mean())

# 2. 產出部門報表
report = df.groupby('Dept').agg({
    'Efficiency': 'mean',
    'Machine_ID': 'count'
}).sort_values('Efficiency', ascending=False)

print("--- OI 營運洞察報表 ---")
print(report)
```
